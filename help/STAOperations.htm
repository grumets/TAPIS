STA tools

This set of tools allows you to perform operations on STA data. Using these tools sends a new request to the STA server. The tools translate the information entered in the dialog boxes and complete the request URL with this data in order to send a new request to the API.

With these tools you can select, filter data, expand related entities from the current entity, sort data, and change the number of records returned, among other things.

Select Columns

Setting Up the Node:

How to Use the Node:

The purpose of the Select Columns tool is to generate a new table containing only the columns selected in the dialog displayed. The selection will be internally applied as a new request to the STA service.

This example contains the original data with all columns:

To display the dialog, double left-click on the node. Below you can see the dialog displayed, showing all the columns from your data. Deselect the columns that you do not want to include in the new table, leaving selected only those you want to keep. After making the selection, press OK button.

This is the result, a new table containing only the selected columns.

Now, the attributes of your data contain only the selected columns. To restore the original ones, you must continue from the previous node.

Take a look at the new request that TAPIS internally makes:

https://citiobs.demo.secure-dimensions.de/staplustest/v1.1/Observations?$select=@iot.selfLink, id, phenomenonTime, resultTime, result, resultQuality, validTime, parameters

Sort by

Setting Up the Node:

How to Use the Node:

Sort By tool allows you to sort data by an attribute (column). Be aware that each request loads 100 records, so the tool will return the first 100 records from the data stored in the STA service. This means that your previous data and the result of the request may include data unrelated to the current context. If you want to sort only your data, you need to ensure that your request returns fewer than 100 records, or use the Sort By tables tool (grey) to work only with the currently loaded data (using the "table" mode).

This example shows data sorted as follows:

To display the dialog, double left-click on the node. Below you can see the dialog, which includes all the columns from your data. Select one column to use as the reference for sorting. You can also choose whether to sort the data in ascending or descending order. After selecting your preferences, click the OK button to apply the changes.

This is the result of the request. In this case, you can see that the data is sorted by result attribute, and the records displayed are not the same as before because the STA service contains more than 100 results for this request.

This is the request sent to the STA service:

https://citiobs.demo.secure-dimensions.de/staplustest/v1.1/Observations?$orderby=resultasc

Record range

Setting Up the Node:

How to Use the Node:

Record range allows you to define a range of records to request from the STA service, indicating from which record to start and how many records you want to receive.

This example shows the current data:

To display the dialog, double left-click on the node. Below you can see the dialog, which includes two fields: one to indicate how many records you want to skip from the beginning of your request, and another to indicate how many records you want to receive. If you want to use this tool to select a number of records different from 100 starting from 0, you must enter 0 in the first field and specify the desired number of records in the second one. After writing your preferences, click the OK button to apply the changes.

This is the result of the request. In this case, you can see that the request has skipped the first 50 records and returned the next 200.

This is the request sent to the STA service: https://citiobs.demo.secure-dimensions.de/staplustest/v1.1/Sensors?$skip=50&$top=200

Count results

Setting Up the Node:

How to Use the Node:

Count Results tool shows you the total number of records you can find in the STA service with your query. TAPIS displays only 100 records by default (or more if you use the Range tool or Entity tool), but this tool returns the total number of all records.

To display the dialog, double left-click on the node. The dialog that appears shows a box where you can enter the interval (in seconds) at which you want to receive a new request for the number of records. By clicking the OK button, you start receiving the information. The counted number will be displayed in TAPIS, below the Count Results node. To stop this automatic refresh, enter the node and click Stop Refresh.

Here you can see the result of the request.

To illustrate the difference, below two different requests have been executed: one asking for all Observations in the STA service (bottom row), and another retrieving only the records from Datastream 20 (using the Record Range tool).

Take a look at this exemple request that TAPIS internally makes (all Observations):

https://citiobs.demo.secure-dimensions.de/staplustest/v1.1/Observations?$count=true
