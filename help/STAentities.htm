STA entities reading tool

This set of tools is used to explore entities in an STA service, so they can only be linked to a node containing STA data. You can only link an entity with a related entity.(To know which entities are linked, consult the schema below)

Any entity can be linked to an STA service node. If you want to explore an entity related to your previous entity, you must ensure that your data contains only one record (by applying the Select row tool, the Filter rows tool, or , with select resource tool or automatically clicking the entity in the table below).

Each time you link an entity,a new request to the API will be made . Every request to the STA service will return 100 records. Double-clicking on this node will display a window where you can enter a number to specify how many records you want to load.

Campaigns

As with any entity, to view the Campaigns of an STA service, this tool must be connected directly to the STA service or to a related entity following the STA schema. If it is connected to another entity, that entity must contain only a single record. To achieve this, you can use the Select row, Filter rows, or Select resource tool.

  • Definition: An event or an activity that results in or is linked to a collection of MultiDatastream, Datastream, and/or ObservationGroup. A campaign is commonly limited in space and time and can be managed by one party.

    Example: A "Water Quality Summer Campaign" carried out during July and August in a river basin, where multiple Datastreams (e.g., pH, temperature, turbidity) are collected by different sensors and grouped under the same campaign.

  • Linked with:Datastreams, MultiDatastreams, Party, License, ObservationGroups

  • Properties:
    • name (string): required
    • description (string): required
    • observationType (string): required
    • unitOfMeasurement (JSON): required
    • observedArea (object): optional
    • phenomenonTime (data_isoperiod): optional
    • resultTime (data_isoperiod): optional
    • properties (JSON): optional

FeaturesOfInterest

If it is connected to another entity, that entity must contain only a single record. To achieve this, you can use the Select row, Filter rows, or Select resource tool.

  • Definition:A FeatureOfInterest is the real-world entity or phenomenon on which an observation is made and to which the result of the observation is assigned. It represents the subject being observed, such as a location, object, or spatial region. In the context of the Internet of Things, the FeatureOfInterest is often the location of the Thing, for example the room where a sensor is installed. In remote sensing scenarios, it can represent a geographic area or volume that is being observed. In short, the FeatureOfInterest is what the observation is about

  • Definition:In the context of the OGC SensorThings API, a MultiDatastream is a type of datastream that groups multiple observed properties measured by a single Thing into one stream, where each Observation returns a single structured result containing multiple values (an array), one for each observed property measured at the same time. For example, a weather station measuring air temperature, relative humidity, and atmospheric pressure would use a MultiDatastream so that each Observation returns a combined result such as [22.5, 61.2, 1013.4], representing all measurements taken simultaneously

Another example is a smartwatch, which is also a Thing because it is a device with a unique identity that can contain sensors like a heart rate sensor and an accelerometer
