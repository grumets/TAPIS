Table tools for data quality

These tools evaluate the quality of your data and provide the results as metadata. This metadata is accumulated together with other metadata that has already been created, making it possible to concatenate quality nodes.

Keep in mind that TAPIS tools only allow you to calculate quality metrics that can be computed automatically. With different tools, you can calculate accuracies, validities, consistencies, omissions, among others.

Some nodes require links to additional nodes besides the one being evaluated.

Logical consistency

Logical consistency checks whether values in a dataset can coexist according to a reference table of valid combinations. It verifies that the relationships between variables follow predefined rules that define which values are allowed to appear together. Temporal quality

Temporal quality evaluates the correctness and reliability of time-related information in a dataset. It ensures that temporal values are accurate, valid, and consistent according to defined rules, such as reference values, valid time ranges, expected resolution, and measurement frequency.

Temporal accuracy:Temporal accuracy is a quantitative indicator of the correctness of time-related data. It is calculated from the dataset, either by averaging an uncertainty column or by computing the standard deviation of time values.

Temporal validity: Temporal validity assesses whether time values conform to a predefined validity range, ensuring that all timestamps fall within acceptable limits.

Temporal resolution: Temporal resolution is a quantitative indicator that measures the proportion of records that satisfy a predefined temporal granularity (e.g., hours, minutes), according to the required resolution of the dataset.

Temporal consistency: Temporal consistency evaluates whether data follows an expected time pattern or frequency. It checks if the time intervals between records match the defined pattern (e.g., hourly), allowing for a small tolerance.

Positional quality

Positional quality evaluates the correctness and reliability of spatial information in a dataset. It ensures that positional values are consistent and meaningful according to defined rules, such as expected precision, measurement conditions, and spatial constraints.

Positional accuracy: Positional accuracy is a quantitative indicator of the correctness of spatial data. It is calculated using the mean of the reported measurement uncertainties when these are available. When no uncertainty information is provided, positional accuracy is computed using the RMSE (Root Mean Square Error) of the repeated position measurements, considering the dispersion of the observed coordinates.

Positional validity: Positional validity assesses whether spatial values fall within predefined spatial limits. It is evaluated by defining maximum and minimum allowable values for the coordinates, and checking whether each position lies within these valid ranges. Any measurement outside these bounds is considered invalid.

Thematic quality

Thematic quality evaluates the correctness and reliability of attribute information in a dataset. It ensures that thematic values are consistent and meaningful according to defined rules, such as expected classifications, measurement uncertainties, and domain constraints.

Thematic accuracy: Thematic accuracy is a quantitative indicator of the correctness of attribute data. When uncertainty values are available, it is calculated using the mean of the values in the uncertainty column. When no explicit uncertainty information is provided, thematic accuracy is derived from the observed attribute values. For numeric attributes, it is computed as the mean standard deviation of the values, reflecting their dispersion. For non-numeric attributes, thematic accuracy is estimated using the relative frequency of the mode (i.e., the most frequently occurring value), representing the level of agreement among observations.

Thematic validity: Thematic validity assesses whether attribute values conform to predefined thematic constraints. For categorical data, this involves checking whether values belong to a defined list of valid categories. For numeric data, validity is evaluated by verifying that values fall within specified minimum and maximum ranges. Any value outside these constraints is considered invalid.
