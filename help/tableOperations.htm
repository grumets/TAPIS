Generic table tools

This set of tools allows you to perform operations on data. These operations are applied to the data already loaded in TAPIS. These tools can also be used with STA data, but you must take into account that from this point you cannot send another request to the STA service and the data will become table-based data.

The operations are applied manually (implemented in the TAPIS code) without sending external requests. You may find some tools that are similar to the STA tools, but in this case the operations are applied to data that has already been loaded.

With these tools you can select, filter, and edit data, create new columns, group data, and perform other operations.

Replace tool allows you to completely or partially replace a word, number, or value. You can apply the replacement to all data or to a specific column. You can replace any kind of data from any type of node. If you want to replace something from an internet request, remember that from this point on, the table will no longer request data from the source. Complete the dialog with your options and click Apply to apply the changes.

This is the CSV table used for the example:

To display the dialog, double left-click on the node. This is how the dialog looks:

Entire table: If you choose this option, the replacement will be applied to all data in the table.

Single column: If you choose this option, the replacement will be applied only to the selected column from the dropdown.

Search and Replace: Enter the text or value to search for in the Search field and the text or value to use as a replacement in the Replace field.

Treat numbers as text: If you check this checkbox, numbers will be treated as text. This means you can partially replace a number. For example, if you enter 23 in the search field, both 23 and 230 will be matched. If you do not treat numbers as text, the entire number must match, so only 23 will be replaced.

Treat dates as text: If you check this checkbox, dates will be treated as text. This means you can parts of a date, similar to how it works with numbers. For example, if you want to replace the year in dates, you need to treat dates as text. If you want to replace a number across the entire table without affecting dates, do not treat dates as text. In that case, the full date must match to be replaced.

Join tables

Setting Up the Node:

How to Use the Node:

The Join Tables tool allows you to merge two tables. This tool can be used with any type of data. The only requirement is that the two tables have at least one corresponding column, containing matching values, regardless of whether the column names are the same.

To perform the join, the tool must be connected to two nodes containing data.

Different types of joins are supported. You can create a table containing only the matching records (Discard), append the matching data from the second table to the first table (Add records of 1st table), or combine both tables into a single result (Add records from both tables).

If both tables contain columns with the same name (other than the join key), the column from the second table will be automatically renamed by appending _table2 to its name. This suffix can later be changed using the Column Meaning tool.

Theses are the tables used in this example:

To display the dialog, double left-click on the node.

This dialog is divided into two sections.

In the upper section (Row Matching Fields), you select the pair(s) of columns that will be used to match records between the two tables. A join can be performed using one or multiple pairs of matching columns, allowing more precise record matching.

The lower section defines the type of join and the resulting output table:

  • Discard: Creates a new table containing only the records that have matching values in both tables (inner join).
  • Add records of 1st table: Keeps all records from the first table and appends the matching columns from the second table whenever a match is found (left join).
  • Join all records: Combines all records from both tables, including those without a matching counterpart (full outer join).

  • Column with a constant value
  • Column with an auto-incremental value starting from an initial value
  • Empty column (you can edit it record by record using the Edit record tool)
This tool can be used with data from your computer (table data) or with a table requested from an internet service. Remember that once you use this tool, you cannot make another request to the server again.

For the example shown, CSV import has been used.
